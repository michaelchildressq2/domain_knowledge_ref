# Source Use Policy

Use source material to learn and synthesize, not to redistribute the source.

## Allowed

- Record bibliographic facts and source IDs.
- Extract concepts, decision criteria, risks, tradeoffs, and implementation moves.
- Write original generalized guidance in the repository's own language.
- Attribute source lineage in frontmatter and source notes.
- Use short quotations only when necessary and within the user's policy.

## Avoid

- Chapter-by-chapter recaps.
- Long quotations or close paraphrase.
- Recreating the sequence, examples, diagrams, tables, or distinctive structure of a source.
- Committing raw documents or extracted text.
- Treating a source author's ideas as your own; name the source lineage clearly.

## Review questions

Before committing, ask:

- Would this file still be useful without access to the source?
- Is it guidance for agents rather than a summary for readers?
- Is the wording original?
- Is source lineage clear?
- Is the source confidence honest?
