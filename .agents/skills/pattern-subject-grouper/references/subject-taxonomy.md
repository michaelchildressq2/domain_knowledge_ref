# Subject Taxonomy

Use these subject areas when grouping architecture, platform, Infrastructure as Code, and data-system pattern files.

## System Qualities

Reliability, scalability, maintainability, operability, correctness, availability, latency, performance, service levels, and user-visible quality goals.

## Change And Delivery

Safe change, delivery pipelines, testing, validation, rollback, roll-forward, compatibility, migration, release strategy, progressive feedback, and deployment workflow.

## Modular Boundaries

Stack design, module boundaries, components, repositories, service ownership, platform clusters, coupling, cohesion, reusable modules, monolith decomposition, and abstraction boundaries.

## Configuration And Contracts

Configuration, parameters, schema evolution, data contracts, message contracts, registries, API/query contracts, compatibility rules, and externalized configuration.

## Data Modeling And Storage

Data models, relational/document/graph choices, query languages, indexes, storage engines, analytical storage, warehouses, and persistence-model fit.

## Replication Consistency And Coordination

Replication, consistency models, transactions, isolation levels, consensus, leader election, conflict resolution, distributed locks, fencing, and distributed-system correctness.

## Partitioning Scaling And Performance

Partitioning, sharding, hot spots, rebalancing, capacity planning, load characterization, throughput, tail latency, and performance trade-offs.

## Pipelines Streams And Derived Data

Batch processing, stream processing, event logs, derived data, materialized views, event-driven integration, dataflows, replay, backpressure, and pipeline data quality.

## Operations Recovery And Governance

Runtime operations, platform engineering, Kubernetes and cluster management, observability, incident response, disaster recovery, data continuity, backups, governance, compliance, workflow metrics, and policy guardrails.

## Security And Secrets

Secrets management, identity, least privilege, authorization, security policy, isolation, tenant boundaries, and sensitive configuration.
